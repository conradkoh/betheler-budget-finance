Version 1.4.0 - Income and Savings Tracking

Features

  • Transaction Types
    • Support for different transaction types: expense (default), income, and savings
    • Specialized UI for each transaction type
    • Filter transactions by type
    • Savings summary on dashboard
    • Income vs. expenses visualization

Technical Details

Backend Implementation

  • Updated schema.ts to add transactionType field in the transactions table
  • Updated existing queries and mutations in services/backend/convex/transactions.ts to support transaction types
  • Created getSavingsSummary query in services/backend/convex/transactions.ts
  • Updated getCategorySummary to filter by transaction type in services/backend/convex/transactions.ts
  • Added validation for transaction types and appropriate schema changes

Frontend Implementation

  • Created TransactionTypeSelect component in apps/webapp/src/components/TransactionTypeSelect.tsx
  • Updated TransactionForm to include transaction type selection in apps/webapp/src/components/TransactionForm.tsx
  • Updated TransactionItem to display transaction types in apps/webapp/src/components/TransactionItem.tsx
  • Updated TransactionList to filter by transaction type in apps/webapp/src/components/TransactionList.tsx
  • Created SavingsSummaryCard component in apps/webapp/src/components/SavingsSummaryCard.tsx
  • Updated CategoryPieChart to include savings and renamed to SpendingBreakdown
  • Added SavingsSummaryCard to dashboard in apps/webapp/src/app/app/page.tsx
  • Updated SpendingGraph to display income vs. expenses in apps/webapp/src/components/SpendingGraph.tsx
  • Added filters for transaction types in transactions page in apps/webapp/src/app/transactions/page.tsx

Implementation Notes: This feature adds support for different transaction types - expense (default), income, and savings. Income transactions do not require a category, while savings are treated as a special transaction type. The dashboard displays a summary of savings and the pie chart includes this information. Users can filter transactions by type and see a more comprehensive financial picture.
